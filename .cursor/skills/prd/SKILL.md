# Overview

Insert overview text here. The agent will only see this should they choose to apply the rule.
Crée une ébauche complète d’un SaaS de sport avec les éléments suivants :

1. Structure / navigation :

Barre de navigation principale avec logo, liens : Accueil, Dashboard, Programmes, Profil, Pricing, Connexion / Inscription.

Dashboard avec menu latéral pour naviguer entre : Mes séances, Historique, Programmes recommandés, Paramètres.

Footer simple avec mentions légales, contact et réseaux sociaux.

2. Pages et contenus :
Page d’accueil :

Hero section impactante : titre clair (“Votre coach sportif personnel”) + sous-texte sur les programmes personnalisés (poids du corps, élastiques, machines).

CTA clair : “Commencez gratuitement”.

Sections : Avantages clés, Témoignages, Aperçu rapide des programmes, FAQ.

Dashboard :

Liste des programmes disponibles avec filtres (zone du corps, matériel, durée).

Vue “Séance du jour” avec timer intégré, vidéo / GIF des exercices, instructions.

Suivi des séances terminées et streaks.

Profil / espace client :

Informations personnelles, niveau, objectifs, matériel disponible.

Historique des séances et programmes suivis.

Connecté à une base de données simple pour stocker : utilisateurs, programmes suivis, statistiques.

Page Pricing :

Présente le modèle Freemium / Pro / Pro+ avec avantages clairement différenciés.

Boutons CTA visibles pour chaque plan.

3. Design et UX :

Style moderne, épuré, UX mobile-friendly.

Palette de couleurs dynamiques mais pas agressives.

Typographie lisible et professionnelle.

Illustrations / icônes sportives cohérentes.

4. Textes & contenus générés :

Rédige textes de présentation, boutons, microcopy (CTA, messages du dashboard, erreurs) adaptés au sport et motivation.

5. Fonctionnalités techniques de base :

Navigation complète fonctionnelle.

Connexion / inscription simulée avec base de données.

Dashboard dynamique : possibilité d’ajouter des séances fictives à l’historique.

Pricing interactif avec switch Free / Pro / Pro+ (simulé, pas de paiement réel nécessaire).

Objectif : obtenir une première version visuelle et navigable du SaaS, prête à être testée et validée avant développement complet.
